## Modifications Made

### Modification 1: Neovim Installation

*Neovim* was installed inside the customized ISO using Cubic’s internal terminal.

Command used:

bash
apt install -y neovim


Neovim is an advanced terminal-based text editor that allows users to edit configuration files and source code directly from the command line.

---

### Modification 2: htop Installation

*htop* was installed as a system process monitoring tool.

Command used:

bash
apt install -y htop


This tool allows users to view CPU usage, memory usage, and active processes in a clearer and more interactive way than the traditional top command.

---

### Modification 3: tree Installation

*tree* was installed as a tool to display files and directories in a tree-like structure.

Command used:

bash
apt install -y tree


This tool makes it easier to review the structure of system directories and project folders.

---

### Modification 4: /etc/skel Customization

The /etc/skel directory was configured so that new users created in the system receive default files and settings.

Commands used:

bash
echo "Bienvenido a FT Linux Project" > /etc/skel/README_PROYECTO.txt
mkdir -p /etc/skel/Proyecto_FT
echo "Distro personalizada creada con Cubic para el proyecto Build, Boot, and Attack." > /etc/skel/Proyecto_FT/info.txt
echo 'alias ll="ls -lah"' > /etc/skel/.bash_aliases


With this modification, any new user will automatically receive a welcome file, a project folder, and a useful alias for listing files from the terminal.

---

## Justification

The modifications were made to provide the customized distribution with useful tools for system administration, development, and file structure analysis.

* *Neovim* was added to make it easier to edit files from the terminal.
* *htop* was included to monitor processes and system resources in a more visual way.
* *tree* was installed to display folder structures in an organized format.
* The */etc/skel* configuration allows the customization to persist for new users, fulfilling the requirement of applying default settings within the customized distro.

These modifications are real, verifiable, and provide useful functionality to the customized system.
